EXPLOIT_BOOM_RA123
1. Summary

On this challenge, we learned about two vulnerabilities on the web: SQL Injection and Cross-Site Scripting (XSS). Also learned how to exploit these vulnerabilities and how to protect our web applications against them.

2. Concepts learned
  • SQL Injection: SQL injection is a type of vulnerability that allows an attacker to execute arbitrary SQL code on the database of a web application, which can lead to data theft, data manipulation, and even complete control of the database.
  • XSS: Cross-Site Scripting (XSS) is a type of vulnerability that allows an attacker to inject malicious code into a web page, which can then be executed by other users who visit the page. This can lead to data theft, session hijacking, and other types of attacks.

DBA_MYSQL_RA12
1. Summary

On this challenge, we learned the differents configurations about mysql/mariadb, and we apllied all the configurations with differents companies.

2. Concepts learned
  • innodb_flush_log_at_trx_commit: This setting controls how frequently InnoDB log buffer data is written to and flushed from disk log files. It provides a balance between data durability, transactional consistency, and write operation performance.
  • innodb_file_per_table: This instructs the InnoDB storage engine to create a separate .ibd file for each table instead of storing all tables in the same tablespace in a single system file. The default setting is ON, and this configuration is recommended because it improves space management and data recovery by allowing tables to be retrieved individually.
